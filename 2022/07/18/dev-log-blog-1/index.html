  • How

    Bearz’s Website 的開發
    前端的基礎技術在 Udemy 的網頁開發課學習得相當完善了，因此在這一階段沒遇到什麼太大的問題，只花了一個下午的時間就將這個網頁寫好了。
    另外提一下，當初 UI 是用 Figma 畫的，好險有新學到這項技能，不然我可能得用 PPT 來畫了。


    將 Bearz’s Website 佈署至 Github Webpage
    本來以為這是有點複雜的技術，沒想到其實超級簡單的。

    1. 先在 Github 新增一個新的 repo，名字格式為: username.github.io
    2. 將整個靜態網頁的資料夾 Push 進這個 repo
    3. 即可在 https://username.github.io/ 看到自己的靜態網頁了

    使用 Hexo 架設 Blog

    • 如何安裝 Hexo
      ※需先在電腦中安裝 Node.js 跟 Git
      在 cmd 執行：npm install -g hexo-cli

    • 如何透過 Hexo 架設 Blog
      在 cmd 執行:
      hexo init <blog_name>
      cd <blog_name>
      npm install
      hexo generate
      hexo server
      即可在 http://localhost:4000/ 看到架好的 Blog

    • 如何更改 Hexo Theme
      選擇想更改的 Theme 樣式，我使用的是許多人使用的 Next。
      先去Next 的 Github將 repo clone 下來放進 blog 資料夾的 themes 中，
      並修改主目錄中的_config.yml 內的theme: landscape改為theme: next，
      即可更改 Hexo Theme。

    將用 Hexo 架好的 Blog 佈署至 Github project page
    這階段是我遇到最多問題的地方，因為同一個 Github 帳號只能有一個.github.io 的網頁，但我想再開一個 Github Page 佈署這個 Blog，該怎麼做呢？
    在網路找了很多方法，但都沒有成功解決，最後發現了 Github project page 這個東西，它只需要在一個新的 repo 中，更改 setting 中的 page 設定就可以另開一個新的 Github page 了。

    而 Hexo 佈署網頁到 Github Page 有更方便的做法，需要先
    npm install hexo-deployer-git --save
    在修改主目錄的_config.yml 中，


    bearzo811 可以替換成自己的 username，blog 可以替換成新開的 repo name
    即可在 https://username.github.io/repo_name/ 看到佈署在 Github Page 的 Blog。

  • Learn

    經過這次的開發，學習到：

    • Github Page 的使用
    • Hexo 框架的應用
    • 同一個 Github 帳戶設立兩個 Github Page

    這次的開發不算太難，花了差不多兩天就完成了，一天寫 Bearz’s Website+佈署上 Github Page，第二天則是用 Hexo 架設 Blog 並佈署上另一個 Github Page 並將它們串連。
    我將它當作暖身操，順便練習寫一篇開發日誌，之後就可以開始進行 Flutter 的應用開發了。
