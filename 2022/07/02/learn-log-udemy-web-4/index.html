Kernel & Shell

  • Kernel is the central part of a computer.
  • Shell is the interface that we interact with Kernel.
    There are two types of shell, GUI and CMD

CMD

Command Prompt
It’s a place that we can directly give commands to the computer, instead of using GUI.

  • cd ___ : Move to target folder.
  • cd .. : back to previous folder.
  • dir : Display file list in a directory.
  • mkdir ___ : create a new folder name ___.
  • type nul > ___.xx: create a ___.xx file.
  • cls : clear CMD’s screen.
  • code ___: open ___ by vsCode.

Unix

  • pwd: show now position.
  • ls : Display file list in a directory.
  • cd ___ : Move to target folder.
  • cd .. : back to previous folder.
  • cd /d : cd to D.
  • mkdir ___ : create a new folder name ___.
  • rmdir ___ : remove a folder name ___. (But this foleder need empty.)
  • rm -rf ___: remove a folder name ___.
  • touch ___.xx: create a ___.xx file.
  • rm ___.xx: remove a ___.xx file.
  • ctrl + L: Clear the screen.
  • code ___: open ___ by vsCode.

Git & GitHub

  • Git is a software that help people with version control.
  • GitHub is an online platform that provides hosting for software development and version control with Git.

    :star: Repository is folder means in GitHub.

Git Command

  • git --version: show your git version.
  • git init: create .git in folder.
  • git config --global user.name "___": set your name.
  • git config --global user.email "___": set your email.
  • git config --list: show your cnfig list.
  • git status : show your git status.
  • git add ___.xx: add ___.xx into Staging Area.
  • git add .: add all file into Staging Area.
  • git add *.html: add all HTML file into Staging Area.
  • git commit -m "what you did in this version": commit file and write what you did in this version.
  • git rm --cached ___.xx: remove ___.xx from Staging Area.
  • git branch ___ : create new branch name ___.
  • git checkout ___: go into branch name ___.
  • git remote add origin ___(github respository url): set remote repository.
  • git push -u origin master: push into github’s repository.
  • git clone ___(github respository url): clone repository in your PC.
  • git push ___(github respository url): push into github’s repository.
  • git pull ___(github respository url): get github’s repository.

Flow

  1. create repository in gitHub.
  2. git clone ___(github respository url).
  3. touch app.js…. wirte your code.
  4. git add .
  5. git commit -m "".
  6. git push ___(github respository url)
